# How-to Guides

**Task-oriented.** Directions that guide the reader through solving a specific real-world problem. Assumes basic competence, focuses on the goal.

- Addresses a specific question/task a competent user already has.
- Series of steps, but can branch/omit unneeded detail.
- No teaching, no explaining background — just how to get it done.

Example candidates: "How to add a new Terraform stack", "How to run a single E2E user story", "How to pin a GitHub Action to a SHA".

See https://diataxis.fr/how-to-guides/
