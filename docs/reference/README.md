# Reference

**Information-oriented.** Technical descriptions of the machinery and how to operate it. Austere, accurate, complete — describes, doesn't explain or teach.

- Structured around the code/system, not the user's task.
- Consistent, dry, precise. State facts.

Example candidates: `just` recipe reference, DynamoDB `users` table schema, API Gateway `/content` endpoint spec, env var / config reference.

See https://diataxis.fr/reference/
