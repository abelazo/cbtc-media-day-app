# Explanation

**Understanding-oriented.** Discussion that clarifies and illuminates a topic — context, background, the "why" behind design choices. Not tied to a task.

- Makes connections, gives context, discusses alternatives and reasoning.
- Opinion and perspective allowed, unlike reference.

Example candidates: "Why the auth header carries `base64(DNI:Name)` instead of a JWT", "Why five Terraform stacks instead of one", "ZIP caching design rationale".

See https://diataxis.fr/explanation/
